Framework Overview
Cascade uses Evidence-Centered Design (ECD) — a principled assessment framework developed by Mislevy, Almond & Lukas (2003) — to connect in-game behavioral data to meaningful estimates of learner competency. ECD structures assessment around three interlinked models.

Competency Model
The competency model defines the knowledge, skills, and reasoning patterns the game is designed to develop and measure. Cascade targets four of the nine systems thinking elements identified by Arnold & Wade (2015), selected for their measurability through behavioral indicators in a single-player simulation context.
CE-1
Causal Reasoning
Identifying and tracking cause-and-effect relationships between system variables — understanding that one variable's change drives changes in others through defined causal pathways.
Failure mode targeted: Players who do not consult the causal structure when making decisions are reasoning from intuition rather than system understanding.
CE-2
Feedback Loop Recognition
Recognizing that system variables form circular causal chains — particularly distinguishing the balancing feedback loop (B1) in which reservoir level constrains sustainable extraction.
Failure mode targeted: Players who over-extract during the drought are treating extraction as an independent decision rather than as a variable constrained by reservoir state.
CE-3
Time-Delay Awareness
Accounting for the lag between an intervention and its downstream effects. In Scenario 1, rainfall effects register one time step after they occur, requiring forward-looking rather than reactive decisions.
Failure mode targeted: Players who respond to drought only after the reservoir visibly drops are reacting to events rather than reasoning about trajectories.
CE-4
Multi-variable Synthesis
Reasoning about multiple interacting variables simultaneously — in Scenario 1, balancing extraction rate against both current reservoir level and incoming rainfall when setting daily extraction.
Failure mode targeted: Players who consult only one variable (e.g., demand) before deciding are not synthesizing the full system state.
📐 Theoretical Grounding

CE-1 Causal Reasoning Causality Event-driven (not stock-and-flow) reasoning Cause-effect identification
CE-2 Feedback Recognition Feedback loops Ignoring accumulation constraints Feedback structure mapping

  Treated output  =  E × η
  Supply deficit  =  max( 0,  D − Treated output )
  Reservoir (t+1) =  clamp( R(t) + Rainfall − E,  0,  100 )

  Where:
    E   = extraction rate (player-controlled: 2 | 4 | 6 | 8 MG/day)
    η   = treatment efficiency = 0.80
    D   = daily demand = 4.0 MG/day
    R   = reservoir level (stock)

  Parameters:
    Reservoir start = 60 MG
    Minimum safe    = 20 MG  (alert threshold, not hard floor)
    Maximum         = 100 MG

Scoring Pipeline
After the final decision is confirmed, the backend applies the evidence rules to the full session's Decision rows and interaction logs. The resulting competency profile drives the radar chart shown on the debrief screen.

Scenarios 2 and 3 will extend these rules to accumulate evidence across multiple nodes for more robust estimates.

Debrief Configuration
Following Kim & Pavlov's (2019) structural debriefing model, the post-scenario debrief translates implicit game experience into explicit systems reasoning vocabulary. Three components are implemented.
Component 1
Decision Timeline
Visual replay of all 8 decisions showing extraction choice per day and whether the causal structure panel was consulted. Surfaces patterns the player may not have been consciously aware of.
Component 2
Causal Map
Player draws the feedback structure of the scenario by placing directed edges with polarity. One edge pre-placed as anchor. Completed map scored against reference model — missing the B1 loop edge indicates feedback recognition not yet internalized.
Component 3
Structured Reflection
Three targeted prompts implementing Chi et al.'s (1989) self-explanation effect: unexpected outcomes, surprising feedback loops, and counterfactual decisions. Responses stored for qualitative analysis in future evaluation.
